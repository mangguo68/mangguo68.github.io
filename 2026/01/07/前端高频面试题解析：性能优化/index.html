前端高频面试题解析：性能优化

一、前端性能优化指标

题目：请解释 Core Web Vitals（核心 Web 指标），以及如何优化这些指标。

答案：
Core Web Vitals 是 Google 提出的衡量用户体验的关键指标，包括 LCP、FID 和 CLS。

解题思路：

  1. LCP (Largest Contentful Paint)：最大内容绘制，衡量首屏加载速度
  2. FID (First Input Delay)：首次输入延迟，衡量交互性
  3. CLS (Cumulative Layout Shift)：累积布局偏移，衡量视觉稳定性

延伸知识点：

  • 优化 LCP：优化服务器响应时间、使用 CDN、优化资源加载
  • 优化 FID：减少 JavaScript 执行时间、使用 Web Workers
  • 优化 CLS：为图片和广告预留空间、避免动态注入内容

二、代码分割与懒加载

题目：什么是代码分割和懒加载？如何实现？

答案：
代码分割是将代码分成多个小块，按需加载；懒加载是在需要时才加载资源。

解题思路：

  1. 代码分割：使用动态导入 import()、webpack 的代码分割配置
  2. 懒加载：图片懒加载、组件懒加载、路由懒加载

延伸知识点：

  • 动态导入返回一个 Promise
  • 路由懒加载可以减少初始包大小
  • 图片懒加载可以使用 Intersection Observer API

三、构建优化

题目：如何优化前端构建过程和构建产物？

答案：
优化构建过程可以提高开发效率，优化构建产物可以提高应用性能。

解题思路：

  1. 构建过程优化：使用缓存、并行构建、增量构建
  2. 构建产物优化：代码压缩、Tree Shaking、代码分割、按需加载

延伸知识点：

  • Tree Shaking：移除未使用的代码
  • 代码压缩：使用 Terser、CSSNano 等工具
  • 模块合并：减少 HTTP 请求

四、资源优化

题目：如何优化前端资源加载？

答案：
资源优化是前端性能优化的重要部分，包括图片、字体、CSS 和 JavaScript 等资源的优化。

解题思路：

  1. 图片优化：使用适当的格式、压缩图片、懒加载
  2. 字体优化：使用系统字体、字体子集、字体预加载
  3. CSS 优化：内联关键 CSS、减少 CSS 文件大小
  4. JavaScript 优化：减少文件大小、使用模块化、按需加载

延伸知识点：

  • 图片格式选择：WebP > JPEG > PNG
  • 字体加载策略：使用 font-display 属性
  • 资源加载顺序：CSS 在前，JavaScript 在后
  • 使用 CDN：加速资源加载

五、渲染性能优化

题目：如何优化前端渲染性能？

答案：
渲染性能优化可以提高页面的响应速度和用户体验。

解题思路：

  1. 减少重排重绘：使用 transform 和 opacity、批量更新 DOM
  2. 使用 CSS 动画：比 JavaScript 动画性能更好
  3. 优化关键渲染路径：内联关键 CSS、延迟加载非关键资源
  4. 使用虚拟列表：处理大型列表

延伸知识点：

  • 重排的触发条件：修改元素位置、大小、内容等
  • 重绘的触发条件：修改元素颜色、背景等
  • 合成层的创建条件：transform、opacity、filter 等
