前端高频面试题解析：浏览器与网络

一、浏览器渲染原理

题目：请解释浏览器的渲染过程，以及如何优化渲染性能。

答案：
浏览器渲染过程包括 HTML 解析、CSS 解析、DOM 构建、CSSOM 构建、渲染树构建、布局、绘制和合成。

解题思路：

  1. 渲染过程：HTML → DOM → CSSOM → 渲染树 → 布局 → 绘制 → 合成
  2. 优化策略：减少重排重绘、使用 CSS 动画、优化关键渲染路径

延伸知识点：

  • 重排（回流）：布局发生变化，性能开销大
  • 重绘：外观发生变化，性能开销小
  • 合成：将绘制的内容合成到屏幕上，性能开销最小

二、跨域问题

题目：什么是跨域？如何解决跨域问题？

答案：
跨域是指浏览器的同源策略限制，当协议、域名或端口不同时，会产生跨域问题。

解题思路：

  1. 同源策略：协议、域名、端口必须相同
  2. 解决方法：CORS、JSONP、代理服务器、postMessage

延伸知识点：

  • CORS 的工作原理：通过 HTTP 头控制跨域访问
  • JSONP 的原理：利用 <script> 标签不受同源策略限制
  • 代理服务器的配置：如 webpack-dev-server 的 proxy 选项

三、HTTP 协议

题目：请解释 HTTP/1.1、HTTP/2 和 HTTP/3 的区别。

答案：
HTTP 协议的不同版本在性能和功能上有很大差异。

解题思路：

  1. HTTP/1.1：持久连接、管道化、分块传输编码
  2. HTTP/2：多路复用、头部压缩、服务器推送、二进制分帧
  3. HTTP/3：基于 QUIC 协议、更好的拥塞控制、连接迁移

延伸知识点：

  • HTTPS 的工作原理：SSL/TLS 加密
  • HTTP 方法：GET、POST、PUT、DELETE、PATCH 等
  • HTTP 状态码：2xx（成功）、3xx（重定向）、4xx（客户端错误）、5xx（服务器错误）

四、缓存策略

题目：请解释浏览器的缓存机制，以及如何设置合理的缓存策略。

答案：
浏览器缓存包括强缓存和协商缓存，合理的缓存策略可以减少网络请求，提高页面加载速度。

解题思路：

  1. 强缓存：通过 Cache-Control 和 Expires 头控制
  2. 协商缓存：通过 Last-Modified/If-Modified-Since 和 ETag/If-None-Match 头控制
  3. 缓存策略：对于静态资源使用长缓存，对于动态资源使用协商缓存

延伸知识点：

  • 缓存位置：Service Worker、Memory Cache、Disk Cache、Push Cache
  • 缓存优先级：Service Worker > Memory Cache > Disk Cache > Network
  • 缓存失效策略：版本号、哈希值

五、浏览器存储

题目：请比较 localStorage、sessionStorage、cookie 和 indexDB 的区别。

答案：
浏览器提供了多种存储方式，每种方式都有其特点和适用场景。

解题思路：

  1. localStorage：持久存储、容量大（约 5MB）、同源共享
  2. sessionStorage：会话存储、页面关闭后清除、容量大（约 5MB）
  3. cookie：小型文本文件、容量小（约 4KB）、随请求发送
  4. indexDB：客户端数据库、容量大、异步操作

延伸知识点：

  • localStorage 的使用场景：存储用户偏好设置、主题等
  • sessionStorage 的使用场景：存储临时会话数据
  • cookie 的使用场景：存储用户身份信息、会话 ID
  • indexDB 的使用场景：存储大量结构化数据
